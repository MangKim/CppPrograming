1. 강의 노트에서 몰랐던 단어
   답 : 궁금했던 단어는 텍스트 에디터 이다. 조사하고 보니 모르는게 이상했던 것인데 그냥 단순하게 글자 데이터만 다루는 도구이다. 추가로 프로그래밍 과정에서 워드 프로세서를 사용하지 않고 텍스트 에디터를 사용하는 이유는 컴파일러는 순수하게 글자만 읽을 수 있기 때문에 워드 프로세서에서 사용하는 글자 색상이나 그런 부분들을 인식하지 못하기 때문에 인식하는 과정에서 에러가 날 수도 있기 때문이다. 
2. 알고리즘 설계와 코딩 중 무엇이 중요하고 그 이유는 무엇인가?
    답 : 알고리즘 설계가 중요하다
    이유 : 알고리즘 설계는 말 그대로 프로그램을 설계하는 과정이기 때문에 설계단계가 더 중요하다. 코딩은 그냥 하나의 번역 과정일 뿐이고 설계가 잘못되는 순간 차후의 과정에서 막대한 자원(시간, 인력 등등)이 소모된다.
3. 클래스와 객체가 무엇인지 설명하시오.
    답 : 클래스는 붕어빵을 찍어내는 붕어빵틀 이라고 할 수 있고 객체는 붕어빵틀로 찍어낸 붕어빵이라고 할 수 있다.
4. 절차지향 프로그래밍 언어와 객체지향 프로그래밍 언어의 차이를 조사하라.
   답 : 절차지향과 객체지향의 차이점으로는 크게 프로그램이 정해진 순서 즉 흐름대로 따라가는 절차지향과 각 객체들이 상호작용하며 프로그램이 실행되는 객체 지향이 있다. 절차지향은 프로그램 실행 속도가 빠른편이고 객체지향은 차후 유지보수가 용이하다는 차이점이 있다.
5. 라이브러리가 무엇인지 자세히 설명하시오.
   답 : 라이브러리는 함수나 클래스들을 미리 작성하여 모아놓은 꾸러미 하고 할 수 있다. 개발자가 모든 기능을 처음부터 만들 필요 없이 만들어진 기능을 가져다 쓸 수 있게 해주는 도구 상자와 같다.
6. 라이브러리가 목적파일 형태로 존재하는 이유를 설명하라.
   답 : 첫번째로는 소스코드를 보호하기 위함이고 두번째는 컴파일 시간을 단축하기 위함이다.
7. 링크과정이 필요한 이유를 설명하라.
   답 : 컴파일 과정만 있으면 각 코드들이 흩어진 낱장의 종이처럼 각 페이지만 번역하는 과정이 이루어지는데 그 페이지들을 하나의 책으로 제본하는 과정이 링크과정이다. 때문에 링크과정이 없으면 프로그램이 잘 작동하지 못하는 것이다.
